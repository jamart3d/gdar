# Issue Report: Screensaver Logo Trails

**App Version:** 1.1.47+148
**Date:** 2026-03-04
**Time:** 20:51

## Issue Summary
The "Logo Trails" feature in the screensaver is implemented but suffers from several architectural and visual inconsistencies that make it feel "broken" or poorly integrated:
1. **Massive Scaling Mismatch**: The trail ghosts are approximately 10x larger than the live logo rendered by the shader.
2. **Audio Pulse Inconsistency**: The live logo (shader) pulses with music bass energy, but the trail ghosts (Dart canvas) are static in size.
3. **Layering Inversion**: The ghosts are rendered on top of the live logo, whereas trails typically appear behind the object.
4. **Buffer Stale-out**: The position buffer stops updating when intensity is 0, causing a "jump" to the current position when first enabled.

## Files Affected
- `lib/steal_screensaver/steal_background.dart`: Scaling and rendering logic for trails.
- `lib/steal_screensaver/steal_game.dart`: Trail buffer management.
- `shaders/steal.frag`: Source of truth for live logo scaling (which Dart currently ignores).

## Potential Causes
- **Inconsistent Scaling Constants**: 
    - Shader uses: `(110.0 / resolution.y) * logoScale`.
    - Dart uses: `logoScale * minDim`. 
    - At 1080p, the shader logo is ~110px tall, while the Dart ghost is ~1080px tall.
- **Missing Energy Mapping**: `StealBackground._renderTrail` does not account for `_beatPulse` or audio energy when calculating `renderSize`, unlike the shader.
- **Draw Order**: `StealBackground.render` calls `_renderTrail` AFTER the shader-painted rectangle.

## Proposed Solutions / Next Steps

### 1. Synchronize Scaling (Critical)
The Dart trail rendering should use the same 110px base constant as the shader to match the live logo's size.
```dart
// Suggested fix in StealBackground._renderTrail
final baseHeight = 110.0; // Match steal.frag constant
final logoRenderSize = config.logoScale.clamp(0.05, 1.0) * baseHeight;
```

### 2. Add Audio Pulse to Ghosts
Multiply the ghost `renderSize` by the current `_beatPulse` (or energy) to make them expand and contract in sync with the live logo.

### 3. Move Trails Behind Logo
The shader should be modified to accept an alpha channel for the background OR the trail ghosts should be rendered *inside* the shader (expensive). Alternatively, using `BlendMode.screen` is a good compromise, but the live logo should ideally be rendered last in a separate `PositionComponent` to ensure it stays on top of the "ghosts".

### 4. Background Buffer Polling
Ensure the trail buffer in `StealGame` updates even when intensity is 0 so that it is always "warmed up" and ready when the feature is toggled on.
