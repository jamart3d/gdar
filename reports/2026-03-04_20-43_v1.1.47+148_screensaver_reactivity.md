# Issue Report: Screensaver Reactivity & Bar Graph Functionality

**App Version:** 1.1.47+148
**Date:** 2026-03-04
**Time:** 20:43

## Issue Summary
The user reported that the screensaver reactivity functionality and the "bar graph" (audio visualizer) are not working as expected. Investigation reveals two primary technical flaws:
1. **Visual Obscuration**: The bar graph component (`StealGraph`) is rendered *underneath* the main screensaver background (`StealBackground`). Since the background shader (`steal.frag`) is entirely opaque (`alpha: 1.0`), the bar graph is 100% hidden from view.
2. **Reactivity Gaps**: 
    - Audio reactivity tuning settings (Peak Decay, Bass Boost, etc.) do not update in real-time if changed while the screensaver is active.
    - The `AudioReactor` is only initialized once in `initState`. If a user enables reactivity in settings and then waits for the screensaver to start, it works; however, if settings are changed via an overlay or external trigger during playback, the reactor does not reset or update its config.

## Files Affected
- `lib/steal_screensaver/steal_game.dart`: Incorrect component stacking order.
- `lib/ui/screens/screensaver_screen.dart`: Lacks live reactivity for audio reactor configuration.
- `shaders/steal.frag`: Opaque output prevents rendering of components underneath the background.

## Potential Causes
- **Flame Component Order**: In `StealGame.onLoad`, `_graph` is added before `_background`. In Flame, later-added components render on top.
- **Opaque Shader**: The `steal.frag` shader always outputs `vec4(finalColor, 1.0)`, which overwrites any previously rendered pixels.
- **State Management**: `ScreensaverScreen` uses `context.watch<SettingsProvider>()` in its build method to construct `StealConfig`, but the internal `AudioReactor` (which is a state member) is not being pushed new configuration values when the build triggers.

## Proposed Solutions / Next Steps

### 1. Fix Rendering Order (Critical)
Swap the initialization order in `lib/steal_screensaver/steal_game.dart` so the graph renders on top of the fluid background.
```dart
// lib/steal_screensaver/steal_game.dart

@override
Future<void> onLoad() async {
  await super.onLoad();
  _subscribeToReactor(_audioReactor);

  // Background first
  _background = StealBackground(config: config);
  add(_background!);

  // Graph second (now it renders on top of the background)
  _graph = StealGraph();
  add(_graph!);
  
  // ...
}
```

### 2. Implement Live Reactor Configuration
Update `lib/ui/screens/screensaver_screen.dart` to push configuration changes to the reactor whenever the `SettingsProvider` updates. Adding a call to `_pushAudioConfig()` within the `build` method (wrapped in a check) or using a `Listener` would solve this.

### 3. Verification
- Test on an Android TV device to confirm the bar graph appears in both 'corner' and 'circular' modes.
- Verify that changing "Bass Boost" or "Peak Decay" in the UI results in an immediate change in visual pulse/decay behavior without restarting the screensaver.
