ok# Issue Report: TV Main Screen UI Regressions

**Report Date:** 2026-04-01 10:58
**App Version:** v1.3.48+258
**Target:** Google TV / Android TV (gdar_tv)

## Issue Summary
Two regressions have been identified on the TV Main Screen:
1. **Typography Failure**: The "Rock Salt" branding font is not being applied to the TV UI, resulting in a fallback to system sans-serif.
2. **Behavioral Inconsistency**: The "PLAY RANDOM SHOW" button in the playback pane's empty state does not match the premium animated sequence and loading feedback triggered by the "Dice" button in the upper-left header.

---

## 1. Font Application Failure ("Rock Salt")

### Investigation Results
Found a case-sensitivity mismatch between the font family registration and the settings configuration.

*   **`packages/shakedown_core/pubspec.yaml`**: The font is registered as `RockSalt`.
    ```yaml
    - family: RockSalt
      fonts:
        - asset: assets/fonts/RockSalt-Regular.ttf
    ```
*   **`packages/shakedown_core/lib/config/default_settings.dart`**: The default font is specified as `rock_salt`.
    ```dart
    static const String appFont = 'rock_salt';
    ```
*   **`packages/shakedown_core/lib/ui/screens/splash_screen.dart`**: Contains a hardcoded override for the title.
    ```dart
    fontKeyOverride: isFruit ? 'rock_salt' : null,
    ```

### Root Cause
Flutter font family lookups are case-sensitive. Because the configuration uses lowercase `rock_salt` while the asset registration uses PascalCase `RockSalt`, the engine fails to locate the font and reverts to the default system font.

## 2. "Play Random Show" Behavioral Inconsistency

### Investigation Results
The "Dice" button (header) and "Play Random Show" button (playback pane) use different widget implementations and trigger the sequence with different levels of visual feedback.

*   **Header (Dice)**: Uses `AnimatedDiceIcon` which provides a rotation/loading animation and is tightly integrated with the `TvDualPaneLayout` sequence.
*   **Playback Pane (Button)**: Uses a standard `TextButton.icon` with `LucideIcons.dice5`.
    ```dart
    // packages/shakedown_core/lib/ui/screens/tv_playback_screen_build.dart:320
    TextButton.icon(
      onPressed: () => context.read<AudioProvider>().playRandomShow(delayPlayback: true),
      icon: Icon(LucideIcons.dice5, color: colorScheme.primary),
      label: Text('PLAY RANDOM SHOW', ...),
    )
    ```

### Root Cause
While both trigger `audioProvider.playRandomShow(delayPlayback: true)`, the `TextButton` lacks the `AnimatedDiceIcon`'s loading state and premium animation. Additionally, `TvDualPaneLayout` imposes a **1.2s delay** (Stage 1) intended for the header dice animation. When triggered from the pane button, this delay feels like a UI "hang" because there is no local animation justifying the pause.

## Proposed Solutions / Next Steps

- Update `DefaultSettings.appFont` to `'RockSalt'`.
- Update `SplashScreen` fontKeyOverride to `'RockSalt'`.
- Audit all uses of `'rock_salt'` string literals across the monorepo and unify them to the registered `'RockSalt'`.

### Step 2: Unify Random Play UI
- Replace the `TextButton.icon` in `tv_playback_screen_build.dart` with a more premium implementation that utilizes `AnimatedDiceIcon`.
- Ensure the "loading" state is reflected in the playback pane button so the 1.2s delay feels intentional.
