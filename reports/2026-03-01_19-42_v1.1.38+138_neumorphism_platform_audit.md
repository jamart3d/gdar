# Neumorphism Implementation Audit: Google TV & Non-Web Platforms

**Date:** 2026-03-01  
**Time:** 19:42  
**App Version:** 1.1.38+138  

## Issue Summary
The goal of this investigation was to determine if Neumorphism (the "Fruit" theme aesthetic) is correctly gated to Web/PWA platforms or if it is leaking into Google TV and other non-web UI environments. 

According to project rules and specs:
- **Fruit Theme/Neumorphism** is strictly for Web/PWA.
- **Google TV** should use a dark-mode focused Android style without heavy translucency or Neumorphic shadows.

## Findings

### 1. Platform Gating Status
The investigation confirms that the core Neumorphic logic is correctly gated by the **Fruit Theme**, which is currently exclusive to Web/PWA (non-TV).

- **`ThemeProvider`**: Correctly forces `ThemeStyle.android` for non-web, TV, or Native Desktop/Mobile environments (lines 85, 125 in `theme_provider.dart`).
- **`NeumorphicWrapper`**: Contains a global guard (line 114) that checks `themeStyle == ThemeStyle.fruit`. 
- **User Settings**: Selection of the "Fruit" style is hidden on all platforms except Web/PWA (non-TV).

### 2. Detailed Platform Audit

| Platform | Style | Neumorphism State | Observation |
| :--- | :--- | :--- | :--- |
| **PWA / Web** | Fruit | **Active** | Primary target for Liquid Glass and Neumorphism. |
| **Google TV** | Android | **Inactive** | Correctly falls back to a high-contrast dark theme without shadows. |
| **Native Mobile**| Android | **Inactive** | No access to Fruit theme settings; uses standard Material 3. |
| **Native Desktop**| Android | **Inactive** | No access to Fruit theme settings; uses standard Material 3. |

### 3. Files Affected & Leakage Risks

| Component | Status | Observation |
| :--- | :--- | :--- |
| **`PlaybackPanel`** | **SAFE** | Explicitly branches to `_buildTvLayout` if `deviceService.isTv` is true, bypassing all Neumorphic code. |
| **`ShowListCard`** | **SAFE** | Uses `isFruit = themeProvider.themeStyle == ThemeStyle.fruit && kIsWeb`. Double-lock ensures zero leakage. |
| **`TvSwitchListTile`** | **LEAK (LOGIC)** | Checks `useNeumorphism` but *not* `isFruit` for its layout padding. While shadows don't render, the padding logic executes on all platforms. |
| **`RatingControl`** | **SAFE** | Uses `isFruitNeumorphic` boolean based on theme style. |
| **`SectionCard`** | **LEAK (LOGIC)** | Checks `useNeumorphism` (line 178) without a theme style gate. Adds redundant `Padding` on TVs/Native apps if settings were somehow to be toggled. |

### 4. Potential Issues Found

- **Redundant Wrappers in `SectionCard`**: Unlike other components that check for `kIsWeb`, `SectionCard` only checks `useNeumorphism` (line 178). Although `NeumorphicWrapper` will return its child if not in Fruit mode, the component still wraps everything in a `Padding` and a potentially empty wrapper, which is slightly less efficient.
- **TV Logic vs. Neumorphism**: In some components like `ShowListAppBar`, the code wraps items in both `NeumorphicWrapper` and `TvFocusWrapper` (line 95). While functionally safe because of the Fruit-gate, it makes the build method more complex than necessary for TV.

## 5. Proposed Solutions / Next Steps

1. **Standardize Gating Logic**: Ensure all components use the `isFruit && kIsWeb` pattern before even attempting to call a Neumorphic or Liquid Glass wrapper. This reduces widget tree depth for TV/Mobile.
2. **Audit `TvSwitchListTile`**: Refactor `tv_switch_list_tile.dart` to ensure the Neumorphic path is strictly for mobile/web and doesn't affect the TV-specific focus logic.
3. **Specs Verification**: Update `fruit_theme_spec.md` to explicitly state that the `isTv` check in `ThemeProvider` is the primary firewall for Neumorphism, and that `NeumorphicWrapper` acts as the secondary guard.

## Conclusion
Neumorphism is **NOT** leaking into the Google TV UI. The system successfully falls back to standard Material/Android styles for TV. However, some shared components could be "cleaned up" to avoid unnecessary widget tree nesting when running on TV.
