# Performance Audit: Android vs Fruit Theme & Simple Mode

## Issue Summary
Investigate the performance characteristics of the Apple-inspired "Fruit" theme versus the standard "Android" (Material 3) theme, specifically focusing on computational intensity, settings interactions, and the mechanics of "Simple Mode" (Performance Mode).

## Performance Intensity Breakdown

| Feature | Android (Standard) | Fruit (Liquid Glass) | Intensity Impact |
| :--- | :--- | :--- | :--- |
| **Glass / Blur** | None (Opaque/Transparent) | `BackdropFilter` (Blur) | **High**: Requires expensive off-screen buffer pass per frame on Web. |
| **Depth** | Single `BoxShadow` (M3) | 4-Layer `Neumorphic` Shadows | **Medium**: Increases paint complexity and draw call overhead. |
| **Animations** | Standard Rises/Fades | Vector Scale Transforms + RGB Gradients | **Medium**: Continuous CPU/GPU synchronization for gradient sweeps. |
| **Borders** | Static hairline | `AnimatedGradientBorder` | **Low/Med**: Constant GPU updates for moving colors. |

### Technical Analysis: Fruit Theme
The **Fruit** theme represents the most intensive UI configuration. 
- **BackdropFilter**: On Flutter Web, `ImageFilter.blur` is notoriously heavy. During music playback, the system automatically reduces blur sigma (e.g., from 15.0 to 8.0) to preserve CPU cycles for the audio engine, but it remains the single largest bottleneck.
- **Neumorphism**: Each `NeumorphicWrapper` generates four distinct shadow objects. For a list like the Show List, this translates to hundreds of additional shadow paint operations.

## Setting Interactions & Constraints

When switching between styles, the system enforces several architectural constraints to maintain aesthetic integrity and performance:

- **Fruit Theme Active**:
    - **Forces Neumorphism**: Enabled by default to provide depth to glass surfaces.
    - **Disables True Black**: True Black is incompatible with "Liquid Glass" since blurs and shadows require a non-zero background color (usually white/black with opacity) to interact with the underlying content.
    - **Disables Dynamic Color**: Uses curated HSL palettes (Sophisticate, Minimalist, Creative) instead of wallpaper-derived colors.
- **Android Theme Active**:
    - **Enables True Black/Dynamic Color**: Returns to standard Material 3 logic.
    - **Disables Neumorphism**: Defaults to standard Material elevation.

## Simple Mode (Performance Mode)

**Simple Mode** is designed as a "Snappy UI" fallback. When enabled, it performs a systematic teardown of intensive graphical passes:

### What happens in Simple Mode?
1. **Blur Extraction**: `LiquidGlassWrapper` replaces the expensive `BackdropFilter` with a simple semi-transparent `Container`. 
    - *Result*: Dramatically reduced GPU usage on Web.
2. **Shadow Stripping**: `NeumorphicWrapper` is completely bypassed, returning only the child widget without any shadow layers.
3. **Visual Simplification**: 
    - **Glow Borders** are forced **OFF**.
    - **Animated Hover Effects** (scale transforms in `ShowListCard`) are disabled.
    - **List Item Padding** is reduced to minimize redraw overlays.

> [!NOTE]
> The **Screensaver** has its own independent **Simple Mode** (Oil Performance Mode) specifically for the shader engine, though this is forced **ON** automatically on Google TV devices.
