# Issue Report: Copy Icon Cut-off in Playback Panel (Fruit Theme)

## Status
- **Issue**: The copy-to-clipboard icon in the `PlaybackPanel` (sliding drawer) appears cut-off, as if it has "haircut" on the top and bottom.
- **Theme**: Fruit Theme (Web UI).
- **Severity**: Low (Visual Bug).
- **Resolution**: **Fixed**. Removed the `Transform.scale` which was blowing the icon up beyond its layout box, replaced it with proper `size` property, and adjusted the Neumorphic container alignment.

## Findings

### 1. Root Cause Analysis
The copy icon is currently using a `Transform.scale(scale: 2.0)` inside an `IconButton`. While this visually enlarges the icon, it does **not** change the layout box of the icon. 

```dart
// lib/ui/widgets/playback/playback_panel.dart (Line 259)

icon: Transform.scale(
  scale: 2.0,
  child: Icon(
      isFruit ? LucideIcons.copy : Icons.copy_rounded,
      size: 20 * scaleFactor,
      color: colorScheme.onSurfaceVariant),
),
```

The `Icon` widget thinks it is $20\text{px}$ (multiplied by scale factor), but `Transform` is blowing it up to $40\text{px}$ rendering-wise. The `IconButton` and its internal `ConstrainedBox` clip the icon because the actual rendered content exceeds the bounds of the widget.

### 2. Neumorphic Interaction
When the Fruit theme is active, `NeumorphicWrapper` and `LiquidGlassWrapper` are applied. These wrappers have their own padding and overflow behaviors, which exacerbate the clipping when the inner icon is artificially scaled beyond its bounds.

## Proposed Solution
We must remove the `Transform.scale` and instead calculate the correct `size` for the `Icon` widget directly. This ensures the Flutter layout engine calculates the correct bounding box and the `IconButton` provides enough space.

## Technical Detail
- **File**: `lib/ui/widgets/playback/playback_panel.dart`
- **Method**: `_buildPhoneLayout` (inside the `IconButton` builder)
- **Change**: 
    - Remove `Transform.scale`.
    - Set `Icon` size to `32 * scaleFactor` (comparable to the desired "large" look).
    - Ensure `padding` on the `IconButton` is sufficient for the Neumorphic glow.
