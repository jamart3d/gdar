# Issue Report: Screensaver Anti-Alias Audit

**App Version:** 1.1.47+148
**Date:** 2026-03-04
**Time:** 20:57

## Issue Summary
The "Anti Alias" setting is implemented using a sub-pixel sharpening technique (`fwidth` + `smoothstep`). While technically functional, it behaves more like an "Edge Sharpening" feature than traditional anti-aliasing, leading to the user's question about its effectiveness. It also has a significant conflict with the "Blur" setting.

## Files Affected
- `shaders/steal.frag`: Source of the sharpening logic and blur conflict.
- `lib/steal_screensaver/steal_background.dart`: Uniform passing logic.

## Potential Causes
1. **Blur Conflict**: The shader samples the *unblurred* texture alpha to calculate the anti-aliased edge (lines 148-151 in `steal.frag`). If a user has "Blur" enabled, the Logo Anti-Alias setting will effectively **force the edges to be sharp**, overriding the desired blur.
2. **Subtle 1-Pixel Ramp**: The `fwidth` technique creates a ramp exactly 1 pixel wide. On a 4K display, a 1-pixel ramp is nearly invisible to the naked eye, making the setting appear "broken" or useless.
3. **Hardened Alpha**: The `smoothstep(0.5 - fw, 0.5 + fw, rawA)` logic assumes the logo is a hard-edged shape. If the logo has intentional soft glows or soft anti-aliasing in the PNG itself, this setting will "crunch" those soft edges into hard ones.

## Proposed Solutions / Next Steps

### 1. Fix the Blur Conflict
The anti-aliasing logic should use the `texColor.a` (which might already be blurred) instead of re-sampling `uTexture`.
```glsl
// Proposed fix in steal.frag
if (uAntiAlias > 0.5) {
    float rawA = texColor.a; // Use the alpha we already calculated (respecting blur)
    float fw = fwidth(rawA) + 0.001; 
    texColor.a = smoothstep(0.5 - fw, 0.5 + fw, rawA);
}
```

### 2. Implementation of "Super-AA" (Optional)
If the goal is to make it look "smoother" on 4K, we could allow the `fw` multiplier to be configurable, effectively creating a "Soft Edge" mode.

### 3. Rename or Explain
If the current behavior (sharpening) is intended for "Fast" performance modes to clean up low-res textures, it should be labeled as "Edge Sharpening" or "CRT Sharpness" rather than Anti-Alias.
