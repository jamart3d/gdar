# TV Premium Highlight Audit Report

**Date**: 2026-03-05
**Version**: 1.1.47+148
**Feature**: Premium TV Highlight (Settings > Interface)

## 1. Executive Summary
The "Premium TV Highlight" feature in version `1.1.47+148` provides a high-fidelity animated gradient border for TV focus states. While visually impressive, the current implementation in `TvFocusWrapper` and `AnimatedGradientBorder` presents a layout overlap issue and potential performance overhead on lower-end TV hardware. The core UI flow and navigation logic remain unchanged and fully functional.

---

## 2. Technical Findings

### 🔴 Layout Issue: Border-Content Overlap
*   **Observation**: `TvFocusWrapper` implements the premium highlight using `AnimatedGradientBorder` with `usePadding: false` and a `borderWidth` of `6.0`.
*   **Root Cause**: In `AnimatedGradientBorder`, the stroke is centered on the widget boundary (`rect.deflate(borderWidth / 2)`). With `6.0` width, this results in a `3.0px` overlap into the child's content area.
*   **Impact**: UI elements like icons or text that sit close to the edge of a `TvListTile` or `TvSwitchListTile` are partially obscured or clipped by the neon gradient.
*   **Recommendation**: Set `usePadding: true` in `TvFocusWrapper` for premium states, or increase the internal margins of the child widgets.

### 🟡 Performance: Constant Repaint
*   **Observation**: The highlight utilizes an `AnimatedBuilder` that triggers a `CustomPaint` repaint every frame while the item is focused.
*   **Context**: `TvFocusWrapper` is used for almost every interactive element on TV (28+ files).
*   **Impact**: While only one item is focused at a time, the constant animation loop remains active. On certain TV SOCs, this can lead to increased GPU load and potential "jank" when scrolling quickly.
*   **Recommendation**: Consider using a `RepaintBoundary` around the `AnimatedGradientBorder` or lowering the frame rate of the glow animation on TV devices.

### 🟢 Flow Verification: Navigation & Interaction
*   **Verification**: Navigation flow (D-Pad) was analyzed by inspecting the `Focus` node implementation in `TvFocusWrapper`.
*   **Result**: The feature is a visual decorator only. The `Focus` node and `GestureDetector` (Tap/Long Press) are located outside the `AnimatedGradientBorder` wrapper.
*   **Conclusion**: Enabling the highlight does not change focus traversal, selection logic, or general UI flow.
