# Issue Report: TV Screensaver Audio Reactivity

**App Version:** 1.1.41+141  
**Date:** 2026-03-02 20:59 PST  
**Reporter:** /issue_report workflow  

---

## Issue Summary

The user asked: *"Does TV UI screensaver audio reactivity work?"*

**Short Answer: Partially — with a critical logic inversion bug that always skips audio reactivity initialization.**

The full Android Visualizer API pipeline (`VisualizerPlugin.kt` → `VisualizerAudioReactor` → `StealGame`) is architecturally correct and well-implemented. However, there is a **control flow inversion** in `screensaver_screen.dart` that means the audio session ID is **never passed to the reactor**, and the real-audio path is only reached when reactivity is **disabled** in settings.

## Root Cause: Logic Inversion in `_initAudioReactor()`

In `screensaver_screen.dart`, lines 98–107:

```dart
// BAD: This block runs when reactivity is DISABLED OR on Web
if (kIsWeb || !settings.oilEnableAudioReactivity) {
  final reactor = await AudioReactorFactory.create(isTv: deviceService.isTv);
  if (mounted) {
    setState(() => _audioReactor = reactor);
    reactor?.start();
  }
  return; // ← RETURNS EARLY, skipping session ID lookup
}

// GOOD: This block only runs when reactivity IS enabled
int? sessionId;
if (defaultTargetPlatform == TargetPlatform.android) {
  final audioProvider = Provider.of<AudioProvider>(context, listen: false);
  sessionId = audioProvider.audioPlayer.androidAudioSessionId;  // ← critical!
}

final reactor = await AudioReactorFactory.create(
  audioSessionId: sessionId,
  isTv: deviceService.isTv,
);
```

The `if` guard at line 99 reads: *"if on Web OR reactivity DISABLED, create a reactor"*. This is backwards. When reactivity is disabled, you should create **no reactor** (or a null one). When it IS enabled on TV, you need to execute the session-ID lookup block below it.

**Result:** When `oilEnableAudioReactivity = true` on TV:
- The factory IS called with a session ID ✅
- But on web/disabled path, the factory is called WITHOUT session ID
- `AudioReactorFactory.create(isTv: true)` with no `audioSessionId` → passes `audioSessionId: null` → native `initialize(0)` → may work but attaches to session 0 (system audio, not app audio)

**When reactivity is disabled on TV:**
- The early-return block fires
- `AudioReactorFactory.create(isTv: true)` is called — returns `VisualizerAudioReactor` without session ID
- Reactor starts but listens to session 0, not the app's audio ❌

## Secondary Issue: `performanceMode` Hardcoded ON for TV

In `screensaver_screen.dart` line 166–167:
```dart
performanceMode: settings.oilPerformanceMode ||
    Provider.of<DeviceService>(context, listen: false).isTv,
```

`performanceMode` is **always forced `true` on TV**, regardless of the user's setting toggle in Screensaver → Performance Mode. This means the screensaver always runs in reduced-complexity mode on TV, even if the device can handle full shaders. The user toggle in the TV settings UI has no effect.

## Proposed Solutions

### Fix 1: Correct the logic inversion (Critical)

```dart
Future<void> _initAudioReactor() async {
  final settings = Provider.of<SettingsProvider>(context, listen: false);
  final deviceService = Provider.of<DeviceService>(context, listen: false);

  // Web never uses the real visualizer
  if (kIsWeb) return;

  // If reactivity is disabled, skip entirely
  if (!settings.oilEnableAudioReactivity) return;

  // Get session ID on Android
  int? sessionId;
  if (defaultTargetPlatform == TargetPlatform.android) {
    final audioProvider = Provider.of<AudioProvider>(context, listen: false);
    sessionId = audioProvider.audioPlayer.androidAudioSessionId;
  }

  final reactor = await AudioReactorFactory.create(
    audioSessionId: sessionId,
    isTv: deviceService.isTv,
  );

  if (mounted) {
    setState(() => _audioReactor = reactor);
    if (reactor is VisualizerAudioReactor) {
      _pushAudioConfig();
    }
    reactor?.start();
  }
}
```

### Fix 2: Respect user's performanceMode setting on TV

```dart
// Remove the forced isTv override — let the user's preference win
performanceMode: settings.oilPerformanceMode,
```

If a TV performance default is desired, set it in `TvDefaults.oilPerformanceMode = true` in `default_settings.dart` so it's a *default* the user can change, not a hardcoded override.

### Fix 3 (Optional): Add a session ID retry
Android audio session IDs can be `null` briefly on startup. Consider adding a short delay or retry if `sessionId == null` before creating the reactor.
