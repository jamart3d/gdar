# Investigation Report: True Black vs. Glow Blur Override

**Date:** 2026-03-01  
**Project:** GDAR Audio Player (v1.1.38+138)  
**Issue:** Investigating whether "True Black" mode overrides "Glow Blur" effects.

---

## 1. Executive Summary
Investigation confirms that **True Black mode is architecturally designed to override Glow Blur (shadows)**. This is enforced both at the UI level (through descriptive text) and at the rendering level (through style computation logic). However, a regression was found in the style utility that makes this override absolute, regardless of user intent.

## 2. Technical Findings

### A. UI Intent
In `AppearanceSection.dart`, the "True Black" toggle explicitly labels itself with the subtitle:
> *"Shadows and blur disabled"*
This confirms that from a design philosophy standpoint, the "Liquid Glass" glows are incompatible with the "True Black" (OLED optimization) goal.

### B. Logic Enforcement
The override is implemented in `CardStyle.compute` (`card_style_utils.dart`):

```dart
// Code in card_style_utils.dart
bool showShadow = settings.glowMode > 0 && (!isTrueBlackMode || settings.glowMode == 2);
```

### C. Logic Bug Identified
The condition `settings.glowMode == 2` in the line above is a **vestigial bug**.
- Recent updates migrated `glowMode` from a discrete index (0, 1, 2, 3) to a percentage (0, 25, 50, 100, etc.).
- Because `glowMode` never equals `2` in the current version, the override `!isTrueBlackMode` is now absolute whenever True Black is enabled. This prevents even a "faint" glow from appearing if desired.

### D. Rendering Result
When `showShadow` is forced to `false` by True Black:
- `AnimatedGradientBorder` skips the `MaskFilter.blur` draw call.
- The user still sees the **Animated RGB Border** (if enabled), but it appears as a sharp, clean line without the diffuse "bloom" or "halo" effect.

## 3. Impact Analysis
- **Benefit:** Ensures perfect `#000000` black levels across the UI by preventing diffuse blurs from "clouding" the OLED pixels.
- **Drawback:** Users who want "True Black" cards but *with* a subtle RGB glow cannot currently achieve this due to the absolute logic override.

## 4. Recommended Fixes

1.  **Refactor Logic**: Update `card_style_utils.dart` to use the new percentage values (e.g., `settings.glowMode >= 50`) instead of the dead `== 2` check.
2.  **UI Feedback**: If "True Black" is ON, the "Glow Border" intensity slider should perhaps be disabled or show a warning that "Blur is disabled, only the border will animate."
