## Recommended Configuration

To ensure the player transitions to and **remains** in high-quality Web Audio mode, use these settings:

| Setting | Recommended Value | Rationale |
| :--- | :--- | :--- |
| **`handoffMode`** | `'immediate'` | Swaps from HTML5 to Web Audio as soon as the first track is decoded (~1-2s). |
| **`backgroundMode`** | `'video'` | Uses a hidden silent video loop to keep the Web Audio context alive in the background. |

## Behavioral Dynamics

### 1. The "Success Path"
When starting a show in the foreground:
1.  **0ms - 2s**: Playback starts via **HTML5** (Instant).
2.  **2s+**: Engine swaps to **Web Audio** (Gapless).
3.  **Background**: If the user hides the tab now, the `video` heartbeat prevents the OS from sleeping the Web Audio context. 
4.  **Result**: 0ms gapless transitions for the remainder of the show.

### 2. Immediate Backgrounding
If the user starts a track and hides the tab **immediately** (< 2s):
1.  The engine stays on **HTML5** for survival.
2.  Restoration to Web Audio is deferred until the **next track boundary** where the tab is visible.
3.  This ensures no clicks, pops, or stalls occur due to OS throttling during a mid-track handoff.
