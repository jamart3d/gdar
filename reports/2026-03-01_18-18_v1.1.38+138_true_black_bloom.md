# Issue Report: Background Bloom in "True Black" Dark Mode

**Date:** 2026-03-01  
**App Version:** 1.1.38+138  
**Environment:** Web (Chrome), Android, `True Black` Mode Enabled

---

## 1. Issue Summary
In "True Black" mode, which is intended for AMOLED screens to achieve pure black (`#000000`) backgrounds, users are reporting a "background bloom" or persistent glowing effect. This breaks the intended high-contrast aesthetic and potentially decreases battery savings for OLED users.

## 2. Files Affected
- `lib/ui/widgets/theme/neumorphic_wrapper.dart`
- `lib/ui/widgets/section_card.dart`
- `lib/main.dart` (Theme initialization logic)

## 3. Potential Causes
The investigation has identified two primary sources of this unwanted "bloom":

### A. Over-aggressive Neumorphic Shadows
The `NeumorphicWrapper` (used extensively in `SectionCard` and settings) generates three layers of shadows to create depth. Even in Dark Mode, it applies:
- A "light shadow" (`Colors.white.withValues(alpha: 0.08)`).
- A "glass highlight" (`Colors.white.withValues(alpha: 0.03)`).
On a pure black background, these subtle white/gray shadows are highly visible and create a halo or "bloom" effect around cards and buttons.

### B. Scaffold Background vs. Surface Mismatch
While `main.dart` correctly sets the `scaffoldBackgroundColor` to `Colors.black`, many widgets (like `SectionCard`) use `Theme.of(context).colorScheme.surfaceContainerHighest.withValues(alpha: 0.3)` as their container color. If the underlying `surface` color is not perfectly black, this alpha blending introduces a faint gray tint that appears as a glow against the pure black scaffold.

---

## 5. Resolution & Implementation
The following fixes have been applied to resolve the bloom issues:

### A. Architectural Enforced Style
- **NeumorphicWrapper**: Now strictly watches both `useNeumorphism` and `themeStyle`. It returns the child immediately (zero shadow pass) if Neumorphism is disabled OR if the theme is NOT `ThemeStyle.fruit`.
- **Appearance Settings**: When switching from `Fruit` to `Android`, the system now explicitly sets `useNeumorphism` to `false`.

### B. True Black Card Hardening
- **SectionCard**: Updated to detect `useTrueBlack && isDark`. In this state:
  - Background color is forced to `Colors.transparent`.
  - A faint `0.2` alpha outline is added to maintain card boundaries without creating light bleed.
  - This eliminates the `surfaceContainerHighest` alpha blending that was causing the diffuse bloom.
