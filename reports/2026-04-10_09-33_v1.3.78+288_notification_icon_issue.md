# Issue Report: Notification Icon is Solid White

**App Version**: 1.3.78+288
**Date**: 2026-04-10
**Status**: Investigated / Solution Proposed

## Issue Summary
The notification icon in the Android status bar appears as a solid white block (silhouette) instead of the intended branded icon. This occurs on modern Android versions (Android 12+ / API 31+) and was first noticed after the monorepo migration.

## Files Affected
1. `packages/shakedown_core/lib/utils/asset_constants.dart`
2. `apps/gdar_mobile/lib/main.dart`
3. `apps/gdar_mobile/android/app/src/main/res/` (multiple directories)
4. `apps/gdar_mobile/android/app/src/main/AndroidManifest.xml`

## Investigation Findings

### 1. Target SDK Enforcement
The application was recently updated to `targetSdk = 35` (Android 15 region). Starting with Android 12 (API 31), the system strictly enforces that notification icons must be monochrome with a transparent background. Any icon that contains non-transparent pixels in non-monochrome colors is automatically silhouetted by the system into a solid white block in the status bar.

### 2. Improper Icon Resource
The current configuration in `packages/shakedown_core/lib/utils/asset_constants.dart` points to the main launcher icon:
```dart
static const String defaultAndroidNotificationIcon = 'mipmap/ic_launcher';
```
Since `ic_launcher` is a full-color app icon (configured via `flutter_launcher_icons` with a background and foreground), it fails the Android monochrome requirement.

### 3. Manifest Inconsistency
In `apps/gdar_mobile/android/app/src/main/AndroidManifest.xml`, the application icon is inconsistently defined:
```xml
android:icon="@mipmap/ic_launcher1"
```
While most resource folders only contain `ic_launcher.png`, `ic_launcher1.png` only exists in `mipmap-xxhdpi`, which may cause fragmentation issues.

## Potential Causes
* **Monorepo Migration Lifecycle**: During the migration to a Workspace structure, the `flutter_launcher_icons` tool was likely rerun, regenerating `ic_launcher` with new branded assets that are not compatible with the notification icon system.
* **SDK Version Jump**: The move to `targetSdk 35` triggered stricter system coloring rules that weren't as apparent on lower API targets.

## Proposed Solutions

### Immediate Fix (Code)
1.  **Create Monochrome Asset**: Generate a new monochrome icon (transparent background, white foreground only) specifically for notifications.
2.  **Add to Resources**: Save the new icon as `ic_stat_shakedown` in `apps/gdar_mobile/android/app/src/main/res/drawable/` resolution folders.
3.  **Update Constants**: Update `AssetConstants.defaultAndroidNotificationIcon` to point to `drawable/ic_stat_shakedown`.

### Design Enhancement
In `apps/gdar_mobile/lib/main.dart`, add the `androidNotificationColor` parameter to `JustAudioBackground.init` to provide a branded tint (e.g., `#0000FF` or a theme-aware blue) for the notification content in the expanded drawer:

```dart
await JustAudioBackground.init(
  // ...
  androidNotificationIcon: 'drawable/ic_stat_shakedown',
  androidNotificationColor: 0xFF2196F3, // Example Blue
);
```

### Hygiene Task
Synchronize the `AndroidManifest.xml` to use `@mipmap/ic_launcher` instead of the non-standard `@mipmap/ic_launcher1` to ensure consistency across all device resolutions.
