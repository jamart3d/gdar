# Issue Report: Current Track Indentation in Fruit Theme

## Status
- **Issue**: Current track title is indented (pushed to the right) compared to other tracks in the music player's track list.
- **Theme**: Fruit Theme (Web UI).
- **Severity**: Low (Visual Inconsistency).
- **Resolution**: **Fixed**. Added a constant placeholder widget to all track items in Fruit/TV modes to lock vertical alignment.

## Findings

### 1. Root Cause Analysis
In the `TrackListView` component, the `ListTile` responsible for rendering each track adds a `leading` widget (a play/pause button) **only** for the currently playing track, and **only** when using the Fruit theme (or TV layout).

```dart
// lib/ui/widgets/playback/track_list_view.dart

if (isPlaying && (isTv || isFruit)) {
  leadingWidget = StreamBuilder<PlayerState>(
    // ... logic to build play/pause icon ...
  );
}
```

When `leadingWidget` is provided to a `ListTile`, Flutter automatically shifts the `title` to the right to make room for it. This creates a visual "indentation" for the playing track. Since non-playing tracks have `leadingWidget` set to `null`, their titles start at the default leading edge of the tile.

### 2. Theme Specificity
This behavior is unique to the **Fruit Theme**. In the default theme, the playing track does not show a leading icon in the list, so all titles remain perfectly left-aligned. 

### 3. Comparison
| Theme | Playing Track | Non-Playing Track | Alignment |
| :--- | :--- | :--- | :--- |
| **Default** | No leading icon | No leading icon | **Aligned** |
| **Fruit** | Play/Pause Icon | No leading icon | **Misaligned (Indented)** |

## Proposed Solution
To maintain perfect alignment while keeping the "Fruit" play button, we should provide an empty placeholder widget (e.g., a `SizedBox` with the same width as the icon) for non-playing tracks in the Fruit theme. 

## Technical Detail
- **File**: `lib/ui/widgets/playback/track_list_view.dart`
- **Method**: `_buildTrackListTile`
- **Current Logic (Lines 369-437)**: Only adds `leading` if playing.
- **Improved Logic**: Always add a `leading` placeholder in Fruit theme, only populate it with the icon if playing.
