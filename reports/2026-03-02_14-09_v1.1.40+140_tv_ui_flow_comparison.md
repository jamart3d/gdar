# Issue Report: Google TV UI Flow Comparison (v1.1.35 vs v1.1.40)

- **Report Date**: 2026-03-02 14:09
- **Baseline Version**: 1.1.35+135 (referred to as "v135")
- **Current App Version**: 1.1.40+140

## Issue Summary
Investigation into the evolution of the **Google TV UI Flow** since Version 1.1.35. While the core "Dual-Pane" architecture remains consistent, significant refinements have been made to remote control efficiency, notification handling, and visual cleanliness.

## Key Changes & Differences

### 1. Remote Control Efficiency (Shortcuts)
- **Then (v135)**: Users were required to navigate focus to specific on-screen buttons (Play/Pause, Seek) to control playback.
- **Now (v1.1.40)**: Implemented **Global Media Key Support**. Hardware remote buttons (`MediaPlayPause`, `MediaNext`, `MediaPrevious`) now trigger `AudioProvider` actions directly from any screen, bypassing the need for manual focus navigation.

### 2. Notification Routing
- **Then (v135)**: System snackbars (errors, buffer alerts) appeared at the bottom of the screen, often obscured by overscan or missed in the large UI layout.
- **Now (v1.1.40)**: Notifications are routed to the `PlaybackMessages` overlay (Top Right). This ensures critical playback status is visible regardless of which pane is active.

### 3. Mini-Player Behavior
- **Then (v135)**: The `MiniPlayer` displayed standard playback controls (Skip/Play) even when browsing a full-screen track list.
- **Now (v1.1.40)**: The `MiniPlayer` now **hides controls** (`hideControls: true`) when displayed on the `TrackListScreen` for TV devices. This reduces visual redundancy since the focus is on the current track list.

### 4. Visual Philosophy & Performance
- **Then (v135)**: Standard Material 3 / Alpha-blended surfaces.
- **Now (v1.1.40)**: While the "Liquid Glass" (Fruit) theme was introduced for Web, the **TV Spec (v1.1)** explicitly guards against `BackdropFilter` and `LiquidGlassWrapper` for performance reasons. TV now uses high-contrast **alpha-blended solid colors** instead of expensive blur effects to maintain 60fps on TV hardware.

### 5. Stars & Badges (Consistency)
- **Analysis**: The logic for showing stars/badges on `ShowListCard` (left pane) remains **identical** to v135. Stars only appear if the show is playing or has a single source. However, the scaling and centering of these elements were refined to prevent the 2.0px overflow issues identified in the recent audit.
