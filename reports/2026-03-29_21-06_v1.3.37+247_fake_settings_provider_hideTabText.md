# Issue Report: missing `hideTabText` in SettingsProvider Mocks

**Date:** 2026-03-29 21:06
**App Version:** 1.3.37+247

## Issue Summary
The `checkup` workflow failed during `melos run test` in the `shakedown_core` package due to two widget test failures.

The root cause of the crash is a `NoSuchMethodError` thrown while building the `FruitTabBar` widget:
`Class 'FakeSettingsProvider' has no instance getter 'hideTabText'.`
and 
`Class '_FakeSettings' has no instance getter 'hideTabText'.`

This indicates that a new getter `hideTabText` was recently added to the `SettingsProvider` contract to control Fruit UI text labels, but the corresponding manual mock and test fake classes were not updated to support it, causing the `FruitTabBar` to crash when it tries to read the setting during pumpWidget.

## Files Affected
**Failing Tests:**
- `packages/shakedown_core/test/ui/screens/fruit_tab_host_race_test.dart`
- `packages/shakedown_core/test/ui/widgets/fruit/fruit_tab_bar_platform_contract_test.dart`

**Mocks to Update:**
- `packages/shakedown_core/test/helpers/fake_settings_provider.dart` (`FakeSettingsProvider` class)
- `packages/shakedown_core/test/ui/widgets/fruit/fruit_tab_bar_platform_contract_test.dart` (`_FakeSettings` class)

## Potential Causes
A recent change to the `FruitTabBar` likely introduced a setting to hide tab text labels (e.g., `hideTabText`), and `SettingsProvider` was updated with the new property. However, because `FakeSettingsProvider` and `_FakeSettings` are manually implemented fakes (rather than auto-generated Mockito stubs), the compiler did not catch the missing implementations until runtime during the widget tests.

## Proposed Solutions / Next Steps
Manually update the fake classes to satisfy the `SettingsProvider` interface.

1. **Update `FakeSettingsProvider`:**
   Open `fake_settings_provider.dart` and add:
   ```dart
   @override
   bool get hideTabText => false; // or appropriate test default
   ```

2. **Update `_FakeSettings`:**
   Open `fruit_tab_bar_platform_contract_test.dart` and add the `hideTabText` getter to the `_FakeSettings` class.

3. **Re-run the tests:**
   Verify the fix by running:
   `cd packages/shakedown_core && flutter test`
   
4. **Trigger /check_mock_parity:**
   Run the `/check_mock_parity` workflow to ensure there are no other missing properties out of sync between `SettingsProvider` and its mocking counterparts.
