# Issue Report: PWA Startup Flicker & Splash Screen Jump

**Date:** 2026-04-05
**Time:** 15:56
**App Version:** v1.3.61+271

## Issue Summary
When launching the application as a PWA on modern mobile devices, there is a visible "jump" or flicker during the transition from the OS-level PWA splash screen to the HTML-based loading splash, and then to the Flutter `SplashScreen` widget. This disrupts the premium "Liquid Glass" aesthetic of the Shakedown (GDAR) web app.

## Files Affected
- [apps/gdar_web/web/index.html](file:///home/jam/StudioProjects/gdar/apps/gdar_web/web/index.html)
- [apps/gdar_web/web/manifest.json](file:///home/jam/StudioProjects/gdar/apps/gdar_web/web/manifest.json)
- [packages/shakedown_core/lib/ui/screens/splash_screen.dart](file:///home/jam/StudioProjects/gdar/packages/shakedown_core/lib/ui/screens/splash_screen.dart)

## Potential Causes

### 1. Manifest vs. HTML Background Color Mismatch
- **manifest.json**: `"background_color": "#000000"`
- **index.html**: `:root { --splash-bg: #080808; }`
- **Result**: A visible background color shift (black to dark gray) occurs as soon as the browser begins rendering the HTML.

### 2. Icon Scaling Discrepancy
- **OS Splash**: Modern Android/iOS PWA engines scale the `manifest.json` icon (usually `Icon-512.png`) to a system-defined percentage of the viewport.
- **HTML Splash**: `index.html` uses `clamp(80px, 20vmin, 160px)` for the image width.
- **Result**: The icon appears to "snap" to a different size (likely shrinking) when the HTML splash div takes over from the system's static splash.

### 3. Transition to Flutter `SplashScreen`
- When Flutter's `flt-glass-pane` is detected, the HTML splash fades out over 600ms.
- The `SplashScreen` widget in Dart has its own layout (Centered `ShakedownTitle` + Checklist) which doesn't visually align with the centered `Icon-512.png` of the previous two layers.

## Proposed Solutions / Next Steps

### Quick Wins (Visual Consistency)
- **Sync Colors**: Update `manifest.json` `background_color` to `#080808` or change `index.html` to `#000000` to eliminate the background flicker.
- **Match Scaling**: Research the standard OS scaling for PWA icons and adjust the `index.html` CSS `width` to better approximate the system splash icon size.
- **Prevent iOS Flicker**: Ensure `apple-mobile-web-app-status-bar-style` matches the background to avoid a white or gray status bar flash.

### Architectural Alignment
- **Converge UI**: Redesign the HTML `loading-splash` in `index.html` to more closely resemble the initial layout of the Flutter `SplashScreen` (e.g., using the "Shakedown" text logo instead of the 512px icon).
- **Reduced Transitions**: Shorten the 600ms fade-out in `index.html` or make the Flutter `SplashScreen` background transparent initially to allow a smoother blend.
