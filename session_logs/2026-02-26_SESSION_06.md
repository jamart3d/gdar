# 2026-02-26_SESSION_06

## Hypotheses
- The `ExpansionTile` inside `SectionCard` rebuilds its children on expand/collapse because its default behavior is to not maintain state. This re-triggers the `HighlightableSetting` pulse animation.
- Deep links to specific settings (like `play_on_tap`) do not get highlighting initially because `_activeHighlightKey` wasn't being set on the first frame if the user didn't trigger a scroll.

## Running Summary
- Added `maintainState: true` to `ExpansionTile` within `lib/ui/widgets/section_card.dart` to prevent children recreation on expand/collapse.
- In `lib/ui/screens/settings_screen.dart`, added `_activeHighlightKey = widget.highlightSetting;` and `_highlightTriggerCount = 1;` in `initState()` when `widget.highlightSetting` is provided.
